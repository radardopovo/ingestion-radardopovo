# Suporte

Use as issues do GitHub para:

- reportar bugs;
- pedir suporte de uso das APIs existentes;
- sugerir novos datasets publicos;
- discutir padroes do monorepo;
- propor melhorias na documentacao.

Ao abrir uma issue, inclua sempre que possivel:

- modulo afetado;
- comando executado;
- periodo ou ano importado;
- erro observado;
- banco de destino e contexto suficiente para reproduzir.
