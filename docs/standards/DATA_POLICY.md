# Politica de dados

- priorize sempre a fonte oficial;
- diferencie dado bruto de dado normalizado;
- nao altere semantica de campo sem documentar;
- deixe o README explicar conversoes relevantes;
- preserve capacidade de auditoria;
- evite esconder descarte de linhas ou correcao automatica sem log.

## O README local deve deixar claro

- de onde o dado veio;
- como foi detectado o arquivo correto;
- quais normalizacoes foram aplicadas;
- quais linhas podem ser ignoradas e por que;
- como o usuario audita o resultado da carga.
